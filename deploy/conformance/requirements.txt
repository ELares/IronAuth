# Python dependencies for the conformance harness, EXACTLY pinned and HASH pinned.
#
# A harness whose stated property is determinism cannot install floating version
# ranges: a range means a rerun months later resolves different code, which is
# the same rot that pinning every image by digest exists to prevent. So every
# package here, including the full transitive closure, is pinned to ONE version
# and ONE artifact hash. Install with --require-hashes so pip REFUSES anything
# whose bits do not match:
#
#     pip install --require-hashes -r requirements.txt
#
# The hashes are for CPython 3.12 on linux x86_64, which is what CI pins with
# actions/setup-python (see .github/workflows/ci.yml). That is deliberate: a hash
# pin is only meaningful against a fixed interpreter and platform. On another
# platform (a developer's macOS box) the wheel legitimately differs, so the
# hashes cannot match there; install the same exact versions without the flag:
#
#     pip install -r requirements.txt
#
# WHO NEEDS WHAT
#   PyYAML    the profile-matrix parser. Needed by the ALWAYS-ON static check
#             lane (scripts/conformance-check.sh) and by run-conformance.sh.
#   requests  the OIDF runner's HTTP client. Needed only to drive the LIVE suite.
# The results gate (parse_results.py) and its unit tests are standard-library
# only and need nothing from this file.
#
# TO BUMP: change the version, then re-resolve the hash with
#     pip download --no-deps --only-binary=:all: \
#         --platform manylinux2014_x86_64 --python-version 3.12 \
#         --implementation cp --abi cp312 <pkg>==<version>
#     pip hash <the downloaded artifact>
